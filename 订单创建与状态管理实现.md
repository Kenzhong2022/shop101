# 订单创建与状态管理实现文档

## 1. 系统架构概述

本系统采用三层订单结构设计：
- **主订单** (orders_master)：用户一次购买的总订单
- **子订单** (order_shops)：按店铺维度拆分的订单
- **订单明细** (order_items)：商品维度的订单项

### 订单流程图

```
购物车 → 创建订单 → 待支付 → 已支付 → 待发货 → 已发货 → 已完成
                ↓
            取消订单/过期
```

```typescript
async function generateMasterOrderNo(): Promise<string> {
  const date = dayjs().format("YYYYMMDD");
  const result = await sql`
    INSERT INTO order_sequence (seq_date, seq_value)
    VALUES (${date}, 1)
    ON CONFLICT (seq_date) DO UPDATE
    SET seq_value = order_sequence.seq_value + 1
    RETURNING seq_value
  `;
  const seq = result[0].seq_value;
  return `${date}${String(seq).padStart(8, "0")}`;
}
```

```typescript
      ${sub.order_status}, ${sub.payment_status}, ${sub.shipping_status},
      ${sub.address_id}, ${sub.remark}, ${sub.expire_at}, ${sub.created_at}
    )
  `);

  // 3. 插入订单明细并扣减库存
  for (const item of sub.items) {
    statements.push(sql`
      INSERT INTO order_items (
        order_shop_id, goods_id, goods_name, quantity, price, total,
        sku_code, sku_value, image, shop_name, shop_id, slave_order_no, user_id
      ) VALUES (
        (SELECT id FROM order_shops WHERE slave_order_no = ${sub.slave_order_no}),
        ${item.goods_id}, ${item.goods_name}, ${item.quantity}, ${item.price}, ${item.total},
        ${item.sku_code}, ${item.sku_value}, ${item.image}, ${item.shop_name}, ${item.shop_id}, ${item.slave_order_no}, ${userId}
      )
    `);

    // 扣减 SKU 库存
    statements.push(sql`
      UPDATE skus
      SET stock = stock - ${item.quantity}, updated_at = NOW()
      WHERE goods_id = ${item.goods_id}
        AND sku_code = ${item.sku_code}
        AND stock >= ${item.quantity}
    `);

    // 扣减商品主表库存
    statements.push(sql`
      UPDATE goods
      SET stock = stock - ${item.quantity}, updated_at = NOW()
      WHERE id = ${item.goods_id} AND stock >= ${item.quantity}
    `);
  }
}

await sql.transaction(statements);
```

## 4. 订单状态流转

### 4.1 支付成功状态更新

```typescript
async function updateOrderStatusByOrderNo(
  orderNo: string,
  updateData: {
    status: string;
    alipayTradeNo: string;
    paidAt: Date;
    buyerId: string;
  },
) {
  const { status } = updateData;
  const statusMap = {
    TRADE_SUCCESS: {
      payment_status: 1,      // 已支付
      order_status: 1,       // 已支付
      item_status: 2,        // 已支付/待发货
    },
  };

  const updateQueries = [
    // 更新主订单
    sql`
      UPDATE orders_master
      SET order_status = ${statusMap[status].order_status}
      WHERE master_order_no = ${orderNo}
    `,
    // 更新子订单
    sql`
      UPDATE order_shops
      SET payment_status = ${statusMap[status].payment_status}, 
          order_status = ${statusMap[status].order_status}
      WHERE master_order_no = ${orderNo}
    `,
    // 更新订单明细
    sql`
      UPDATE order_items
      SET item_status = ${statusMap[status].item_status}
      WHERE order_shop_id IN (
        SELECT id FROM order_shops WHERE master_order_no = ${orderNo}
      )
    `,
  ];

  await sql.transaction(updateQueries);
}
```

### 4.2 过期订单取消

```typescript
async function cancelOrderTransaction(masterOrderNo: string, sql: any) {
  const orderItems = await sql`
    SELECT goods_id, sku_code, quantity
    FROM order_items
    WHERE order_shop_id IN (
      SELECT id FROM order_shops WHERE master_order_no = ${masterOrderNo}
    )
  `;

  const queries = [
    // 更新主订单状态为已取消
    sql`
      UPDATE orders_master 
      SET order_status = 5, updated_at = NOW() 
      WHERE master_order_no = ${masterOrderNo}
    `,
    // 更新子订单状态
    sql`
      UPDATE order_shops 
      SET order_status = 5, updated_at = NOW() 
      WHERE master_order_no = ${masterOrderNo}
    `,
    // 更新订单明细状态
    sql`
      UPDATE order_items 
      SET item_status = 5, updated_at = NOW() 
      WHERE order_shop_id IN (
        SELECT id FROM order_shops WHERE master_order_no = ${masterOrderNo}
      )
    `,
  ];

  // 恢复库存
  for (const item of orderItems) {
    queries.push(sql`
      UPDATE skus
      SET stock = stock + ${item.quantity}
      WHERE goods_id = ${item.goods_id} AND sku_code = ${item.sku_code}
    `);
  }

  await sql.transaction(queries);
}
```

## 5. 定时任务：取消过期订单

### 5.1 任务配置

```typescript
export default defineTask({
  meta: {
    name: "cancel-expired-orders",
    description: "检查并取消过期的订单",
  },
  async run() {
    const sql = getNeon();
    const LOCK_ID = 2026031901;
    const BATCH_SIZE = 500;

    // 1. 获取分布式锁
    const lockResult = await sql`SELECT pg_try_advisory_lock(${LOCK_ID})`;
    const isLocked = lockResult[0].pg_try_advisory_lock;

    if (!isLocked) {
      return { result: "success", updatedCount: 0, skipped: true };
    }

    // 2. 查询过期订单
    const nowIso = new Date().toISOString();
    const orderResult = await sql`
      SELECT master_order_no 
      FROM orders_master 
      WHERE order_status = 0 
        AND expire_at <= ${nowIso}  
      ORDER BY expire_at ASC        
      LIMIT ${BATCH_SIZE}
    `;

    // 3. 批量处理过期订单
    for (const order of orderResult) {
      await cancelOrderTransaction(order.master_order_no, sql);
    }
  },
});
```

### 5.2 锁机制

使用 PostgreSQL 的 `pg_try_advisory_lock` 实现分布式锁，防止多个实例同时执行任务。

## 9. 关键技术点

### 9.1 事务处理

使用 PostgreSQL 事务确保数据一致性：
- 订单创建时，主订单、子订单、订单明细、库存扣减必须全部成功或全部回滚
- 支付成功时，同时更新三个表的状态

### 9.2 分布式锁

使用 `pg_try_advisory_lock` 实现分布式锁，防止定时任务并发执行。

### 9.3 订单号生成

基于 sequence 表生成有序订单号，格式为：`YYYYMMDD + 8位序号`

### 9.4 过期时间处理

订单创建时设置 30 分钟过期时间，定时任务定期检查并取消过期订单。

## 12. 注意事项

1. **库存一致性**：订单创建和取消时必须保证库存的原子性操作
2. **订单过期**：使用定时任务定期检查过期订单，避免占用库存
3. **并发控制**：使用分布式锁防止定时任务并发执行
4. **状态同步**：支付成功后需要同步更新主订单、子订单、订单明细三个表的状态
5. **错误处理**：订单创建失败时需要回滚所有操作，包括已扣减的库存
